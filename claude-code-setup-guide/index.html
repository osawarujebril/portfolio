SECTION 0: Windows Setup — Read This First If You're on Windows

WINDOWS USERS: START HERE Claude Code was originally built for Mac/Linux. On Windows, it needs Git Bash to work. If you skip this section, you WILL get this error:

Claude Code on Windows requires Git Bash. If installed but not in the path, set environment variable pointing to your bash.exe: CLAUDE_CODE_GIT_BASH_PATH=C:\Program Files\Git\bin\bash.exe
Follow every step below IN ORDER. Do not skip any step.

Mac users — skip to Section 1.

1 Install Node.js (Required)

Claude Code runs on Node.js. You need version 18 or higher.

Download and Install:

  1. Go to nodejs.org
  2. Click the LTS (Long Term Support) button — this downloads the Windows installer (.msi file)
  3. Run the installer
  4. Click Next through all the screens. IMPORTANT: On the "Tools for Native Modules" screen, check the box that says "Automatically install the necessary tools" if it appears
  5. Click Install and wait for it to finish

Verify it worked:

Open Command Prompt (press Win + R, type cmd, press Enter) and type:

node --version

You should see something like v20.11.0 or higher. If you see an error, restart your computer and try again.

Also check npm (Node's package manager):

npm --version

Should show a version number like 10.2.0.

If node --version shows "not recognized" Node wasn't added to your PATH. Restart your computer first. If it still doesn't work, uninstall Node.js, re-download, and during installation make sure the "Add to PATH" checkbox is checked.

2 Install Git (with Git Bash) — THIS IS THE KEY STEP

This is where most people get stuck. Claude Code on Windows requires Git Bash — it's a Linux-like terminal that comes bundled with Git.

Download and Install:

  1. Go to git-scm.com
  2. Click "Download for Windows" — it will auto-detect 32-bit or 64-bit
  3. Run the installer (.exe file)
  4. CRITICAL SCREENS during installation — pay attention to these:
Installation Screen: "Adjusting your PATH environment" You will see 3 options. Select the SECOND option:

( ) Git from Git Bash only
(x) Git from the command line and also from 3rd-party software <-- SELECT THIS ONE
( ) Use Git and optional Unix tools from the Command Prompt
This adds Git (and Git Bash) to your system PATH — which is exactly what Claude Code needs.
Installation Screen: "Choosing the default editor" Select "Use Visual Studio Code as Git's default editor" if you see it. If not, any option is fine — this doesn't affect Claude Code.
Installation Screen: "Configuring the terminal emulator" Select: "Use Windows' default console window"
(Either option works, but this one causes fewer issues.)
  1. Click Next through the remaining screens (defaults are fine)
  2. Click Install and wait for it to finish
  3. IMPORTANT: Check "Launch Git Bash" at the end to make sure it opens

Verify it worked:

Open a NEW Command Prompt (close any old ones first) and type:

git --version

Should show something like git version 2.44.0.windows.1

Now check that bash.exe exists where Claude Code expects it:

dir "C:\Program Files\Git\bin\bash.exe"

You should see the file listed. If you see "File Not Found," try:

dir "C:\Program Files (x86)\Git\bin\bash.exe"
Write down which path worked Whether it's C:\Program Files\Git\bin\bash.exe or C:\Program Files (x86)\Git\bin\bash.exe — you'll need this path in the next step.

3 Fix the Git Bash PATH Error — The Actual Fix

This is the error you saw. Here's how to fix it permanently:

Method A: Set the Environment Variable (Recommended — Permanent Fix)

  1. Press Win + R, type sysdm.cpl, press Enter
  2. Click the "Advanced" tab
  3. Click "Environment Variables..." button at the bottom
  4. Under "User variables" (top section), click "New..."
  5. Fill in:
    • Variable name: CLAUDE_CODE_GIT_BASH_PATH
    • Variable value: C:\Program Files\Git\bin\bash.exe
  6. Click OK on all three windows to save
  7. RESTART VS Code completely (close it and reopen it)
Visual Step-by-Step:
Win + R → type "sysdm.cpl" → Enter
→ "Advanced" tab
→ "Environment Variables..." button
→ Under "User variables" → click "New..."
→ Variable name: CLAUDE_CODE_GIT_BASH_PATH
→ Variable value: C:\Program Files\Git\bin\bash.exe
→ OK → OK → OK
→ RESTART VS CODE

Method B: Quick Terminal Fix (Temporary — For Testing)

If you just want to test quickly before doing Method A, open Command Prompt and run:

set CLAUDE_CODE_GIT_BASH_PATH=C:\Program Files\Git\bin\bash.exe

Then launch VS Code from THAT SAME terminal:

code .
Method B only lasts until you close the terminal. Always do Method A for a permanent fix. Method B is just for quick testing.

Method C: PowerShell Alternative (Also Permanent)

Open PowerShell as Administrator and run:

[System.Environment]::SetEnvironmentVariable(
  "CLAUDE_CODE_GIT_BASH_PATH",
  "C:\Program Files\Git\bin\bash.exe",
  "User"
)

Then restart VS Code.

Verify the fix worked:

Open a NEW Command Prompt and type:

echo %CLAUDE_CODE_GIT_BASH_PATH%

Should print: C:\Program Files\Git\bin\bash.exe

If it prints %CLAUDE_CODE_GIT_BASH_PATH% (the variable name itself), the variable wasn't saved. Redo Method A.

The WAT Framework

Everything runs on three layers:

3 Set Up the .claude/ Configuration

The .claude/ folder is Claude Code's brain. Three files control everything:

File 1: .claude/settings.json

This controls what Claude Code is allowed to do:

{
  "permissions": {
    "allow": [
      "Read(~/**)"
    ]
  }
}
What This Does Gives Claude read access to your files. Claude Code will ask permission for other operations (writing files, running scripts, etc.) the first time — you can approve them as they come up, or add more permissions here later.

File 2: .claude/settings.local.json

This controls which websites Claude can fetch from (for research, extraction, etc.):

{
  "permissions": {
    "allow": [
      "WebFetch(domain:YOUR-ALLOWED-DOMAIN-1.com)",
      "WebFetch(domain:YOUR-ALLOWED-DOMAIN-2.com)"
    ]
  }
}

Replace with actual domains you want Claude to access. Or leave empty and approve domains as they come up.

File 3: .claude/mcp.json (MCP Server Config)

This connects Claude Code to external automation platforms:

{
  "mcpServers": {
    "trigger.dev": {
      "command": "npx",
      "args": ["trigger.dev@4.4.0", "mcp"]
    }
  }
}
What's MCP? Model Context Protocol — it lets Claude Code talk to external services. Trigger.dev is a workflow automation platform (like Zapier, but you write code). If you don't need automation, you can skip this file for now.

7 Create Your CLAUDE.md (The Brain File)

The CLAUDE.md file is the MOST IMPORTANT file in the entire setup. It lives in your project root and tells Claude Code:

  • What framework to use (WAT)
  • Where everything is located
  • What skills are available
  • Rules and behaviors to follow
  • Current project status
Think of CLAUDE.md as the Instruction Manual Without it, Claude Code has skills but doesn't know when or how to use them. With it, Claude Code knows EXACTLY what to do for any request.

Structure of CLAUDE.md:

# YOUR PROJECT NAME — Master Navigator

Section 6: How the System Works

The WAT Framework in Practice

Here's how a real task flows through the system:

Example: "Create a Facebook ad for my webinar"
  1. Claude reads CLAUDE.md → sees that ad copy tasks should use the ads/ skill
  2. Claude checks if copy-master should run first → yes, for any copy task, copy-master runs first (12-step intelligence system)
  3. copy-master produces a brief → identifies offer, researches market, diagnoses awareness level, selects emotional arc and framework
  4. ads skill takes the brief → generates 3 ad variations with different hook types
  5. If the ad needs an image → Claude routes to nano-banana-images/ or canvas-design
  6. Result: Complete ad package — copy + visuals — ready to post

Key Operating Rules

  1. Look for existing tools first. Check tools/ before building anything new.
  2. Learn and adapt when things fail. Read error, fix script, retest. Document what you learned.
  3. Keep workflows current. When you find better methods, update the workflow.
  4. Self-improvement loop: Identify break → fix tool → verify → update workflow → move on stronger.
  5. API keys in .env only. Never hardcode. Never log secret values.

Skill Routing — How Claude Decides What To Use

CLAUDE.md contains an "Invoke When" column for every skill. Claude matches your request to the right skill automatically. Examples:

  • "Write a sales page" → copy-master (first) → sales-page
  • "Create webinar slides" → webinar-slides
  • "Build a landing page" → landing-page
  • "Generate a realistic photo" → nano-banana-images
  • "Extract intelligence from this book" → book-extraction
  • "Set up a cron job" → trigger-automation

Section 8: Troubleshooting & FAQs

Q: Claude doesn't know about my skills

Fix: Check that CLAUDE.md exists in your project root AND lists the skills. Claude reads CLAUDE.md on every conversation start.

Q: A skill says "file not found"

Fix: The skill expects files in specific folder paths. Make sure your folder structure matches Section 5 exactly. Run the mkdir command from Step 2 again.

Q: API calls are failing

Fix: Check your .env file. Make sure the key names match exactly (no extra spaces). Test the key manually if possible.

Q: Python tools won't run

Fix: Run pip install python-pptx Pillow requests python-dotenv. Make sure Python 3.10+ is installed: python3 --version

Q: MCP server won't connect

Fix: Make sure Node.js is installed (node --version). The MCP config uses npx which requires Node. Restart VS Code after adding mcp.json.

Q: Claude isn't following the rules I set

Fix: Rules go in CLAUDE.md. Make sure they're clear, specific, and not contradicting each other. Claude re-reads CLAUDE.md each conversation.

Q: I want to add a new skill

How: Use the skill-builder skill. Tell Claude: "Create a new skill for [what you need]." It will generate the SKILL.md with proper structure.

Q: How do I update an existing skill?

How: Edit the SKILL.md file directly, or use skill-builder to audit and optimize it.
